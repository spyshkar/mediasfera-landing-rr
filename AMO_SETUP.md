# AmoCRM + TG + Email — чеклист подключения

## Что нужно изменить перед деплоем

| Параметр | Где менять | Что поставить |
|---|---|---|
| `TG_CHAT_ID` | `.env.php` | ID группы РР-лендинга (не конференции) |
| `AMO_PIPELINE_ID` | `.env.php` | ID нужной воронки |
| `AMO_STATUS_ID` | `.env.php` | ID нужного этапа |
| `SITE_ORIGIN` | `.env.php` | Реальный домен на Beget |

## 1. Настройка AmoCRM

- Войти как администратор
- Настройки → Интеграции → создать или использовать существующую интеграцию
- Получить `access_token`
- Записать домен: `mediasfera.amocrm.ru`

> Текущий токен (от конференции) живёт до **августа 2026**.  
> Тот же токен можно использовать, достаточно поменять воронку/этап.

## 2. Найти ID воронки и этапа

Открыть в браузере:
```
https://mediasfera.amocrm.ru/api/v4/leads/pipelines
```
(с заголовком Authorization: Bearer TOKEN)

Найти нужную воронку и нужный первый этап, скопировать их `id`.

## 3. Настройка Telegram

- Создать новую группу для РР-заявок (или использовать существующую)
- Добавить бота в группу
- Получить ID группы через `@getmyid_bot` или API

## 4. Деплой на Beget

1. Создать папку на хостинге (например, `/rr/`)
2. Загрузить `index.html` и `submit.php`
3. Создать `.env.php` на сервере (не загружать через git!)
4. Проверить форму тестовой заявкой

## 5. Тест submit.php

```bash
curl -s -X POST https://ВАШ-ДОМЕН/submit.php \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","email":"test@test123.com","consent":true}'
```

Ответ: `{"ok":true}`

## 6. Частые ошибки

| Симптом | Причина | Решение |
|---|---|---|
| Нет заявок ни в TG, ни в AMO | `.env.php` не создан | Создать файл на сервере |
| AMO `401` | Токен истёк | Обновить `AMO_TOKEN` |
| AMO `400` | Неверный pipeline_id | Проверить ID через API |
| TG: сообщений нет | Бот не добавлен в группу | Добавить и дать права |
| CORS error в браузере | `SITE_ORIGIN` не совпадает | Проверить домен без `/` на конце |
